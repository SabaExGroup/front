FxTwitter / VxTwitter troubleshooting (server)

https://api.fxtwitter.com is an API endpoint, not a website — an empty browser response is normal.

Test from your server:

curl -sS "https://api.fxtwitter.com/elonmusk" | head -c 200
curl -sS "https://api.vxtwitter.com/elonmusk" | head -c 200
Result Likely cause Action
FxTwitter timeout / refused Datacenter IP blocked Deploy latest backend — VxTwitter fallback runs automatically
Both fail Outbound firewall Enable proxy (Settings → proxy)
Both fail — Fill twitterUrlPool or set xBearerToken

Provider order is configured under Settings → Integrations → Runtime → xTwitter (profileProviderBaseUrls, one URL per line).

VxTwitter-only mode skips broad symbol typeahead — only slug candidates are looked up, then pool / synthetic fallback.
